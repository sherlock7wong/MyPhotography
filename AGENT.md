# AGENT.md

## 项目定位

这是 WonG 的个人摄影作品集网站，用来展示摄影作品和个人风格。

网站当前以 `SherlockWonG` 为品牌名，主要展示这些分类：

- `Portrait`：人像作品
- `Landscape`：风景作品
- `Life`：生活记录
- `CityScape`：城市景观

这个项目不是复杂框架项目，目前主要由原生 HTML、CSS、JavaScript 和 Node.js 本地服务组成。

## 当前项目结构

- `index.html`：网站首页。
- `portrait/index.html`：人像分类页。
- `landscape/index.html`：风景分类页。
- `life/index.html`：生活分类页。
- `cityscape/index.html`：城市景观分类页。
- `src/styles.css`：网站主样式。
- `src/main.js`：前台交互和内容加载逻辑。
- `admin/index.html`：后台管理页面。
- `admin/admin.css`：后台样式。
- `admin/admin.js`：后台登录、编辑、上传和保存逻辑。
- `data/content.json`：网站可编辑内容数据。
- `uploads/`：后台上传的摄影图片。
- `assets/`：网站默认 SVG 图片和占位资源。
- `server.js`：本地 Node.js 服务，负责静态文件、登录、内容保存和图片上传接口。
- `package.json`：启动脚本，目前使用 `npm start` 运行 `node server.js`。
- `启动网站.bat` 和 `start-server-hidden.vbs`：Windows 本地启动辅助文件。

## 协作规则

回答 WonG 时必须使用清楚、直白、容易听懂的话。

不要只给结论，要说明做了什么、为什么这样做、怎么验证。

如果需求明确，直接执行。
如果会影响页面结构、数据格式、登录权限、上传文件或已有作品内容，必须先说明影响，再动手。

不要擅自扩大任务范围。
WonG 只要求改一个文件时，只改那个文件。确实需要改多个文件时，先说明原因。

## 文件安全规则

不要乱删文件。

没有 WonG 明确同意，不得删除、移动、重命名这些内容：

- `uploads/`
- `data/content.json`
- 各分类页目录
- `server.js`
- `admin/`
- `src/`
- 启动脚本文件

不要清空 `uploads/`。这里存放的是已上传的摄影作品图片。

不要随手生成多个版本、备份文件或额外说明文件。
如果 WonG 只要求创建一个文件，就只创建一个文件。

## 代码修改规则

修改前先看相关文件和上下文，不要只根据文件名猜实现。

优先做最小可行改动。
不要为了“更优雅”重构无关代码。

项目当前没有外部依赖。
新增依赖前必须说明原因，并确认现有原生 HTML、CSS、JavaScript 或 Node.js 内置模块无法合理完成。

前台内容主要由 `data/content.json` 驱动。
修改作品数据时，要保持这些关系一致：

- 首页作品窗口使用 `home-01` 到 `home-18`
- 分类页作品窗口使用类似 `portrait-01`、`landscape-01`、`life-01`、`cityscape-01` 的 ID
- HTML 中的 `data-window-id` 必须能在 `data/content.json` 找到对应数据
- 首页筛选分类目前使用 `portrait`、`landscape`、`city`

图片路径优先使用：

- `/uploads/文件名`
- `assets/文件名`

不要把本地绝对路径直接写进网页内容。

## 内容和文案规则

这是摄影作品集网站，文案要克制、清楚、有摄影气质，不要写成营销广告。

不要擅自改变 WonG 已确认的品牌名、邮箱、作品分类和作品标题。

如果只是修文字，不要顺手改页面结构。
如果只是改结构，不要顺手重写文案。

## 编码和中文显示

当前部分后台文件和接口提示里存在中文乱码痕迹。
不要在无明确要求时大范围重写或转换编码。

如果任务是修复乱码，必须：

- 先说明会影响哪些文件
- 保持文件为 UTF-8
- 修完后验证后台页面、登录提示、保存提示和上传提示是否正常显示

## 本地运行和验证

启动方式：

```bash
npm start
```

或：

```bash
node server.js
```

默认访问地址：

- 前台首页：`http://localhost:8080/`
- 后台管理：`http://localhost:8080/admin/`

如果修改了前台页面，需要检查：

- 首页是否正常显示
- 四个分类页是否正常显示
- 图片是否能加载
- 移动端导航是否还能打开和关闭
- 首页筛选按钮是否正常工作

如果修改了后台或接口，需要检查：

- 后台是否能打开
- 登录状态接口是否正常
- `/api/content` 是否能读取
- 保存内容是否会更新 `data/content.json`
- 上传图片是否会写入 `uploads/`

## 安全注意事项

`server.js` 中包含后台登录相关逻辑。
涉及账号、密码、Cookie、上传、文件路径和接口权限时，必须特别谨慎。

不要把 `server.js`、`package.json`、`data/`、`.git/` 或其他敏感文件暴露给前台访问。

上传接口只应接受图片文件。
不要放宽上传类型，除非 WonG 明确要求并理解风险。

## 汇报格式

每次完成修改后，向 WonG 简要说明：

- 做了什么
- 改了哪些文件
- 如何验证
- 还有什么风险或需要 WonG 决定

如果没有运行验证，必须明确说没有运行，并说明原因。
